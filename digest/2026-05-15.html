论文方向一：MoE 推理效率与压缩：从路由策略到无损压缩

拟定论文题目：面向稀疏 MoE 大语言模型的动态路由剪枝与高阶拓扑约束压缩方法

论文核心问题：如何在保持模型性能的前提下，通过动态专家路由剪枝和拓扑感知的无训练压缩，显著降低稀疏 MoE 架构的推理计算开销。

创新点一：基于二值掩码的 Token 自适应动态专家路由剪枝

大多数现有方法采用固定的 Top-K 路由策略激活专家，导致大量冗余计算和次优推理延迟，且在高稀疏度下性能急剧下降 [1, 22, 27]。少数现有方法尝试通过架构修改或重训练实现动态路由，但往往需要昂贵的重新训练成本，或引入训练-推理不匹配问题，难以在实际部署中落地 27。为此，本文提出基于可训练二值掩码的 Token 自适应专家选择方法（BEAM），通过直通估计器和辅助正则化损失实现端到端训练，在不改变原始模型架构的前提下诱导动态专家稀疏性，保留超过 98% 的原始性能同时将 MoE 层 FLOPs 降低高达 85%。

  • 创新类型：方法设计创新
  • 创新性判断：该创新点针对固定 Top-K 路由的冗余计算问题，提出了一种无需架构修改、可端到端训练的二进制掩码动态路由方法，与现有方法相比在稀疏度和性能保持之间取得了更好的平衡。

创新点二：基于高阶拓扑覆盖的无训练 MoE 压缩方法

大多数现有无训练 MoE 压缩方法依赖成对专家之间的相似性或 KL 散度进行合并，忽略了三个专家之间可能存在的不可约循环结构，导致压缩后的模型出现不可预测的性能损失 [22, 27]。少数现有方法开始关注高阶结构，但缺乏系统的数学框架来识别和解决三元组级别的合并障碍 22。为此，本文提出 HodgeCover 方法，通过构建以专家为顶点、KL 合并障碍为边、三元组障碍为面的 2-复形，利用单纯拉普拉斯算子的调和核精确识别不可约循环，并采用贪心覆盖策略优先处理调和关键边和三元组关键面，结合幸存者的权重剪枝，在三个开源稀疏 MoE 骨干网络上实现更优的压缩-性能权衡。

  • 创新类型：问题建模创新 + 方法设计创新
  • 创新性判断：该创新点首次将高阶拓扑障碍形式化为单纯拉普拉斯算子的调和核，并基于此设计贪心覆盖压缩策略，解决了现有成对方法无法处理三元组不可约循环的根本性缺陷。

- 评价指标：模型性能（困惑度、准确率）、FLOPs 减少率、推理延迟、压缩率
- 消融实验：不同稀疏度下的性能保持、动态路由与静态路由的对比、高阶覆盖与成对覆盖的对比

论文方向二：LLM 推理与部署的系统级鲁棒性：攻击、不匹配与资源核算

拟定论文题目：面向 LLM 推理部署的系统级鲁棒性：从加速坍塌攻击到训练-推理不匹配诊断与全链路能耗核算

论文核心问题：如何系统性地识别和缓解 LLM 推理部署中由机制漏洞、实现差异和资源核算缺失引发的鲁棒性问题。

创新点三：基于草稿-目标模型不匹配的投机解码加速坍塌攻击

大多数现有方法将投机解码视为一种安全的加速技术，主要关注如何提高草稿接受率，而忽略了草稿模型与目标模型之间固有的近似误差可能被利用作为攻击面 2。少数现有方法开始关注投机解码的安全性，但尚未系统性地研究如何通过微小扰动同时降低草稿接受率并保持目标模型输出语义不变 2。为此，本文提出 Mistletoe 攻击方法，通过联合优化一个降低草稿-目标一致性的退化目标和一个保持语义的约束目标，在保持目标模型可见行为不变的前提下，将平均接受长度从 3.5 降至 1.2，实现隐蔽的加速坍塌。

  • 创新类型：方法设计创新（攻击方法）
  • 创新性判断：该创新点首次揭示了投机解码中草稿-目标不匹配作为攻击面的机制漏洞，并提出了一种隐蔽的加速坍塌攻击方法，填补了投机解码安全性研究的空白。

创新点四：训练-推理不匹配的诊断与缓解框架

大多数现有 LLM 强化学习系统假设 rollout 生成与策略优化两个阶段的 token 概率完全一致，忽略了实现差异导致的数值分歧 3。少数现有方法注意到这种不匹配现象，但将其视为无害的数值噪声，未能独立分析其对训练稳定性的影响 3。为此，本文在零不匹配诊断环境 VeXact 中隔离 TIM，证明微小的 token 级数值分歧可以独立引发训练崩溃，改变有效优化问题，并提出一套缓解方案，将 TIM 从二阶噪声提升为一阶稳定性因素。

  • 创新类型：问题建模创新 + 方法设计创新
  • 创新性判断：该创新点首次将训练-推理不匹配从数值噪声重新定义为系统级扰动，并通过隔离实验证明其独立引发训练崩溃的机制，提出了针对性的缓解方案。

论文方向三：Agent 工作流与工具编排：拓扑、路由与自适应执行

拟定论文题目：面向 LLM Agent 工作流的拓扑分类、延迟-质量路由与自适应分布式执行

创新点五：基于认知功能与执行拓扑的二维 Agent 架构分类框架
