一、三篇论文方向与六个创新点

论文方向一：LLM Agent 系统鲁棒性与可靠性评估

拟定论文题目： 面向工具编排与推理-执行不匹配的 LLM Agent 系统鲁棒性评估与诊断方法

论文核心问题： 如何系统性地识别、量化和诊断 LLM Agent 在工具调用、编排开销、训练-推理不匹配等场景下的系统性失效模式，并构建可诊断、可复现的评估方法与基准。

创新点一：编排开销与工具调用失效的系统性诊断创新

大多数现有方法将 LLM Agent 的最终任务准确率作为主要评估指标，忽略了工具编排过程中引入的操作性失效模式，如超时、回溯、工具调用失败等，导致评估结果无法反映系统瓶颈 11016。少数现有方法开始关注工具调用过程中的结构有效性，但通常将编排开销视为次要因素，缺乏对编排开销与最终性能之间因果关系的系统性诊断 61821。为此，本文提出一种基于编排开销分解的诊断评估框架，将 Agent 执行轨迹中的工具调用失败、回溯次数、超时事件等编排开销指标与最终任务准确率进行因果关联分析，以量化编排开销对系统鲁棒性的影响。

  • 创新类型： 评估方法创新 + 诊断框架创新
  • 创新性判断： 文献池中 1 关注链式升级任务评估但未诊断编排开销，10 关注微服务鲁棒性测试但未聚焦 Agent 编排，16 通过负消融实验揭示编排开销但未提出系统性诊断方法。本文创新点在于将编排开销从噪声提升为可诊断的系统性指标。

创新点二：训练-推理不匹配（TIM）的鲁棒性影响评估创新

大多数现有方法在评估 LLM Agent 系统时假设训练和推理阶段的模型行为一致，忽略了由于实现差异导致的训练-推理不匹配（TIM）对系统鲁棒性的影响 589。少数现有方法开始关注 TIM 问题，但通常将其视为数值噪声而非系统性扰动，缺乏对 TIM 在 Agent 系统中传播和放大的评估方法 3423。为此，本文提出一种 TIM 敏感性评估方法，通过构造零不匹配诊断环境，量化 TIM 在 Agent 工具调用链中的传播效应，并建立 TIM 敏感度指标以评估 Agent 系统在不同实现差异下的鲁棒性。

  • 创新类型： 评估指标创新 + 鲁棒性分析方法创新
  • 创新性判断： 文献池中 5 首次系统研究 TIM 问题但聚焦于 RL 训练稳定性，8 关注 MoE 路由的训练-推理不匹配，9 关注预填充-解码阶段的效率不匹配。本文创新点在于将 TIM 评估引入 Agent 系统鲁棒性评估领域。

论文方向二：大模型持续学习与记忆架构的自适应演化

拟定论文题目： 面向长期交互场景的 LLM 记忆架构自演化与遗忘诊断方法

论文核心问题： 针对 LLM 在持续预训练、长期对话、多会话交互中的灾难性遗忘与人格漂移问题，设计无回放、自演化的记忆与参数更新架构，并建立可诊断的遗忘评估方法。

创新点三：无回放持续预训练中的遗忘诊断与预警创新

大多数现有方法依赖回放缓冲区、任务标识符或正则化惩罚来缓解持续预训练中的灾难性遗忘，这些方法在 LLM 规模下扩展性差，且无法在遗忘发生前提供预警信号 31529。少数现有方法开始探索无回放持续学习方法，但通常缺乏对遗忘过程的早期诊断能力，无法区分可逆的短期遗忘与不可逆的长期退化 2417。为此，本文提出一种基于轨迹级前兆信号的遗忘诊断方法，通过监测锚点熵收缩、表示漂移冻结和尾部覆盖侵蚀三个前兆指标，在标准验证指标退化之前发出遗忘预警，以支持及时干预。

  • 创新类型： 诊断方法创新 + 预警机制创新
  • 创新性判断： 文献池中 3 首次提出静默崩溃概念并识别前兆信号，15 提出无回放持续预训练方法但缺乏诊断机制，4 研究量化对遗忘的影响但未涉及预警。本文创新点在于将静默崩溃的前兆信号应用于持续预训练的遗忘诊断。

创新点四：自演化记忆架构的闭环优化创新

大多数现有记忆系统将检索基础设施视为固定不变的，仅更新存储内容而保持评分函数、融合策略和答案生成策略在部署后冻结，导致记忆系统无法适应交互模式的变化 172713。少数现有方法开始探索记忆架构的自适应调整，但通常依赖人工配置或离线调优，缺乏自动化的闭环演化机制 31523。为此，本文提出一种自演化记忆架构，将完整的检索配置暴露为结构化动作空间，由 LLM 驱动的诊断模块在每轮演化中读取失败日志、识别根因并提出配置调整，同时配备自动回退和停滞探索保护机制，以实现记忆系统的持续自适应优化。

  • 创新类型： 系统架构创新 + 闭环优化机制创新
  • 创新性判断： 文献池中 17 提出自演化记忆架构但聚焦于检索配置演化，27 提出时间记忆治理框架但缺乏自演化能力，13 提出离散买家画像学习但未涉及记忆架构演化。本文创新点在于将记忆架构的自演化从检索配置扩展到完整的闭环优化。

论文方向三：高效推理与模型压缩的协同优化

拟定论文题目： 面向 MoE 路由与量化部署的协同推理优化方法

论文核心问题： 针对 MoE 路由冗余、量化部署中的遗忘逆转、多适配器服务效率低下等问题，从系统层与算法层协同降低推理延迟与存储开销，同时保持模型能力。

创新点五：动态专家路由与量化感知的协同压缩创新

大多数现有方法将 MoE 路由优化和模型量化视为独立问题，分别优化专家选择策略和权重压缩方案，忽略了二者之间的交互影响 7825。少数现有方法开始探索路由与量化的联合优化，但通常采用后处理方式，在量化后重新调整路由策略，导致优化效果受限 4926。为此，本文提出一种路由-量化协同优化框架，在训练或微调阶段同时学习动态专家掩码和量化感知的路由策略，通过端到端的联合优化实现高稀疏度下的模型能力保持。

  • 创新类型： 方法设计创新 + 协同优化机制创新
