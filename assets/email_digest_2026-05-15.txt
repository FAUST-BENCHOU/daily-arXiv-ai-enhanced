1. **MoE 推理效率与压缩：从路由策略到无损压缩** — 围绕稀疏 MoE 架构的推理效率瓶颈，从专家路由冗余（BEAM 二值掩码）、无训练压缩中的高阶拓扑障碍（HodgeCover）到规模化超参数稳定性（μP 理论），探索不牺牲性能的加速与压缩路径。
2. **LLM 推理与部署的系统级鲁棒性：攻击、不匹配与资源核算** — 聚焦推理阶段系统层面的脆弱性：投机解码的加速坍塌攻击（Mistletoe）、训练-推理数值不匹配导致的训练崩溃（TIM）、蒸馏管线的全链路能耗核算，以及函数调用异步化（AsyncFC）带来的延迟优化。
3. **Agent 工作流与工具编排：拓扑、路由与自适应执行** — 面向 LLM Agent 的工程化挑战：从认知功能与执行拓扑的二维分类框架、功能等价工具提供商的延迟-质量路由（LQM-ContextRoute），到可并行化 Agent 工作流的分布式架构（APWA）与端到端强化学习自动多智能体系统（MetaAgent-X）。

## 论文方向一：MoE 推理效率与压缩：从路由策略到无损压缩

**拟定论文题目**：面向稀疏 MoE 大语言模型的动态路由剪枝与高阶拓扑约束压缩方法

**论文核心问题**：如何在保持模型性能的前提下，通过动态专家路由剪枝和拓扑感知的无训练压缩，显著降低稀疏 MoE 架构的推理计算开销。

### 创新点一：基于二值掩码的 Token 自适应动态专家路由剪枝

大多数现有方法采用固定的 Top-K 路由策略激活专家，导致大量冗余计算和次优推理延迟，且在高稀疏度下性能急剧下降 [1, 22, 27]。少数现有方法尝试通过架构修改或重训练实现动态路由，但往往需要昂贵的重新训练成本，或引入训练-推理不匹配问题，难以在实际部署中落地 [27](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.14438)。为此，本文提出基于可训练二值掩码的 Token 自适应专家选择方法（BEAM），通过直通估计器和辅助正则化损失实现端到端训练，在不改变原始模型架构的前提下诱导动态专家稀疏性，保留超过 98% 的原始性能同时将 MoE 层 FLOPs 降低高达 85%。

- **创新类型**：方法设计创新
- **创新性判断**：该创新点针对固定 Top-K 路由的冗余计算问题，提出了一种无需架构修改、可端到端训练的二进制掩码动态路由方法，与现有方法相比在稀疏度和性能保持之间取得了更好的平衡。

### 创新点二：基于高阶拓扑覆盖的无训练 MoE 压缩方法

大多数现有无训练 MoE 压缩方法依赖成对专家之间的相似性或 KL 散度进行合并，忽略了三个专家之间可能存在的不可约循环结构，导致压缩后的模型出现不可预测的性能损失 [22, 27]。少数现有方法开始关注高阶结构，但缺乏系统的数学框架来识别和解决三元组级别的合并障碍 [22](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.13997)。为此，本文提出 HodgeCover 方法，通过构建以专家为顶点、KL 合并障碍为边、三元组障碍为面的 2-复形，利用单纯拉普拉斯算子的调和核精确识别不可约循环，并采用贪心覆盖策略优先处理调和关键边和三元组关键面，结合幸存者的权重剪枝，在三个开源稀疏 MoE 骨干网络上实现更优的压缩-性能权衡。

- **创新类型**：问题建模创新 + 方法设计创新
- **创新性判断**：该创新点首次将高阶拓扑障碍形式化为单纯拉普拉斯算子的调和核，并基于此设计贪心覆盖压缩策略，解决了现有成对方法无法处理三元组不可约循环的根本性缺陷。

**该论文两个创新点之间的关系**：创新点一解决“如何动态选择专家以减少冗余计算”，创新点二解决“如何在无训练条件下安全合并专家以进一步压缩”。二者属于同一论文主题（MoE 推理效率优化），但分别对应路由阶段和压缩阶段，逻辑上形成递进关系：先通过动态路由减少激活专家数量，再通过拓扑感知压缩合并剩余专家，不存在重复。

## 论文方向二：LLM 推理与部署的系统级鲁棒性：攻击、不匹配与资源核算

**拟定论文题目**：面向 LLM 推理部署的系统级鲁棒性：从加速坍塌攻击到训练-推理不匹配诊断与全链路能耗核算

**论文核心问题**：如何系统性地识别和缓解 LLM 推理部署中由机制漏洞、实现差异和资源核算缺失引发的鲁棒性问题。

### 创新点三：基于草稿-目标模型不匹配的投机解码加速坍塌攻击

大多数现有方法将投机解码视为一种安全的加速技术，主要关注如何提高草稿接受率，而忽略了草稿模型与目标模型之间固有的近似误差可能被利用作为攻击面 [2](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.14005)。少数现有方法开始关注投机解码的安全性，但尚未系统性地研究如何通过微小扰动同时降低草稿接受率并保持目标模型输出语义不变 [2](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.14005)。为此，本文提出 Mistletoe 攻击方法，通过联合优化一个降低草稿-目标一致性的退化目标和一个保持语义的约束目标，在保持目标模型可见行为不变的前提下，将平均接受长度从 3.5 降至 1.2，实现隐蔽的加速坍塌。

- **创新类型**：方法设计创新（攻击方法）
- **创新性判断**：该创新点首次揭示了投机解码中草稿-目标不匹配作为攻击面的机制漏洞，并提出了一种隐蔽的加速坍塌攻击方法，填补了投机解码安全性研究的空白。

### 创新点四：训练-推理不匹配的诊断与缓解框架

大多数现有 LLM 强化学习系统假设 rollout 生成与策略优化两个阶段的 token 概率完全一致，忽略了实现差异导致的数值分歧 [3](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.14220)。少数现有方法注意到这种不匹配现象，但将其视为无害的数值噪声，未能独立分析其对训练稳定性的影响 [3](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.14220)。为此，本文在零不匹配诊断环境 VeXact 中隔离 TIM，证明微小的 token 级数值分歧可以独立引发训练崩溃，改变有效优化问题，并提出一套缓解方案，将 TIM 从二阶噪声提升为一阶稳定性因素。

- **创新类型**：问题建模创新 + 方法设计创新
- **创新性判断**：该创新点首次将训练-推理不匹配从数值噪声重新定义为系统级扰动，并通过隔离实验证明其独立引发训练崩溃的机制，提出了针对性的缓解方案。

## 论文方向三：Agent 工作流与工具编排：拓扑、路由与自适应执行

**拟定论文题目**：面向 LLM Agent 工作流的拓扑分类、延迟-质量路由与自适应分布式执行

**论文核心问题**：如何系统性地分类、优化和并行化 LLM Agent 的工作流与工具编排，以提升复杂任务下的执行效率和鲁棒性。

### 创新点五：基于认知功能与执行拓扑的二维 Agent 架构分类框架

大多数现有 Agent 架构框架仅从单一视角描述系统：工业指南关注执行拓扑（数据流如何流动），而认知科学调查关注认知功能（Agent 做什么）[15](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.13850)。少数现有方法尝试结合两个视角，但缺乏系统的跨轴分析来证明正交性，导致架构上不同的系统（如同一种 Orchestrator-Workers 拓扑可以实现 Plan-and-Execute、Hierarchical Delegation 或 Adversarial Verification 三种模式）被归为同一类 [15](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.13850)。为此，本文提出一个二维分类框架，结合认知功能轴（7 个类别：上下文工程、记忆、推理、行动、反思、协作、治理）和执行拓扑轴（6 个结构原型：链、路由、并行、编排、循环、层次），形成 7x6 矩阵识别出 27 个命名模式，并通过系统跨轴分析证明正交性。

- **创新类型**：指标体系创新 + 分类方法创新
- **创新性判断**：该创新点首次提出认知功能与执行拓扑的二维正交分类框架，解决了现有单一视角框架无法区分架构上不同但拓扑相同的 Agent 系统的根本问题。

### 创新点六：基于延迟-质量匹配的上下文感知工具提供商路由方法

大多数现有工具增强型 LLM Agent 假设每个工具类型只有一个提供商，忽略了通过多个功能等效提供商访问同一工具类型时的提供商路由问题 [17](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.14241)。少数现有方法引入上下文赌博机路由，但通常允许低延迟抵消低质量答案的负面影响，导致路由决策偏离质量优先原则 [17](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-15&paperId=2605.14241)。为此，本文提出 LQM-ContextRoute，一种延迟-质量匹配的上下文赌博机路由器，通过按服务周期内的预期答案质量对提供商进行排序，结合查询特定质量估计和 LLM 作为评判的反馈，在保持延迟-质量前沿的同时，在主要网络搜索负载基准上将 F1 提升 2.18 个百分点。

- **创新类型**：方法设计创新 + 约束处理创新
- **创新性判断**：该创新点首次提出延迟-质量匹配原则用于功能等效工具提供商路由，解决了现有方法中延迟与质量在加性奖励中相互抵消的问题，实现了更优的延迟-质量权衡。
