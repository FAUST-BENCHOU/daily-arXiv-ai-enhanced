# 每日硬凑师兄要求的创新点 2026-05-11

## 今日三个主题（结构化）
1. **LLM 推理效率与缓存管理** — 围绕 KV Cache 压缩、淘汰策略与稀疏注意力索引，探索端到端可学习的缓存淘汰（LKV）、输出感知的全局淘汰框架（LaProx）以及将稀疏注意力建模为范围搜索问题的方法，以降低长上下文推理的内存与计算开销。
2. **智能体系统路由、治理与可靠性** — 聚焦 LLM 智能体的工具调用路由（Switchcraft）、运行时约束治理架构（SARC）、推理可靠性对齐（CASPO）以及早期经验驱动的路由策略（BoundaryRouter），解决智能体部署中的成本、合规与可信问题。
3. **分布式系统资源调度与性能建模** — 涵盖 LLM 推理服务中的分支并行度调节（TAPER）、AI-RAN 异构资源分层调度（HAF）、配置无关的性能剖析（Dooly）以及生成式推荐系统的分布式 KV 缓存加速（RcLLM），面向在线服务的动态资源优化。

## 三篇论文方向与六个创新点

### 论文方向一：面向长上下文推理的 KV Cache 压缩与淘汰机制

**拟定论文题目：** 面向长上下文推理的端到端可学习 KV Cache 压缩与全局淘汰方法

**论文核心问题：** 如何设计一种端到端可学习的 KV Cache 压缩框架，在保持推理精度的同时，通过全局视角和输出感知的淘汰策略，显著降低长上下文推理的内存与计算开销。

#### 创新点一：端到端可学习的 KV Cache 压缩框架

大多数现有方法依赖启发式规则（如注意力权重、频率统计）进行 KV Cache 压缩，这些方法基于统计先验而非任务目标，导致资源分配不当 [13](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06676)[16](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07234)[26](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06763)。少数现有方法开始引入可学习机制，但通常将预算分配与 Token 选择分离处理，缺乏端到端的联合优化 [13](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06676)[26](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06763)。为此，本文提出一个端到端可学习的 KV Cache 压缩框架，将头级预算分配与 Token 级重要性选择整合为统一的优化问题，通过可微的代理损失函数直接对齐压缩策略与下游任务目标，以消除启发式代理与真实目标之间的偏差。

- **创新类型：** 方法设计创新
- **创新性判断：** 池内文献 [13](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06676)（LKV）提出了端到端可学习的 KV 淘汰，但本文进一步将预算分配与 Token 选择整合为统一优化，并引入输出感知的全局重要性度量，与 [13](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06676) 形成差异化。

#### 创新点二：输出感知的全局淘汰策略

大多数现有 KV 淘汰方法基于头级局部注意力权重进行决策，忽略了值表示、输出投影和跨头交互对 Token 贡献的影响 [13](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06676)[16](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07234)[26](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06763)。少数现有方法开始建模注意力图与值状态的交互，但通常仍采用头级局部决策，缺乏全局可比的 Token 重要性度量 [16](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07234)。为此，本文提出一种输出感知的全局淘汰策略，显式建模注意力图与投影值状态的乘法交互，为所有 Token 分配全局可比较的重要性分数，实现模型级统一选择而非头级局部决策，以更准确地保留对最终输出贡献最大的 Token。

- **创新类型：** 方法设计创新
- **创新性判断：** 池内文献 [16](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07234)（LaProx）提出了输出感知的淘汰策略，但本文进一步将全局重要性度量与端到端可学习框架结合，实现模型级统一选择，与 [16](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07234) 形成互补。

### 论文方向二：面向智能体系统的路由、治理与可靠性对齐

**拟定论文题目：** 面向智能体系统的成本感知路由与运行时治理架构

**论文核心问题：** 如何在智能体系统中实现成本感知的模型路由，同时通过运行时治理架构确保工具调用的合规性与可靠性。

#### 创新点三：成本感知的智能体工具调用路由

大多数现有模型路由器针对聊天补全设计，不适用于智能体工具调用场景，且通常忽略工具调用的 Token 密集型推理带来的额外成本 [12](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07180)[15](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07112)。少数现有方法开始针对工具调用设计路由器，但主要依赖静态分类器，缺乏对早期行为经验的利用和冷启动场景的适应 [15](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07112)。为此，本文提出一种成本感知的智能体工具调用路由框架，利用早期行为经验构建紧凑的记忆库，在推理时检索相似案例指导路由决策，同时将工具调用的 Token 消耗纳入成本模型，以在保证正确性的前提下最小化推理成本。

- **创新类型：** 方法设计创新
- **创新性判断：** 池内文献 [15](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07112)（Switchcraft）和 [12](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07180)（BoundaryRouter）分别提出了工具调用路由和早期经验路由，但本文首次将两者结合，并显式建模 Token 消耗成本，形成差异化贡献。

#### 创新点四：运行时约束治理架构

大多数现有智能体系统的治理控制附着于提示词、仪表盘或事后文档，约束在执行后才被评估，造成结构性不匹配 [3](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07728)[4](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06890)。少数现有方法提出运行时治理架构，但通常依赖有限奖励惩罚替代硬约束，且未扩展到多智能体工作流 [3](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07728)。为此，本文提出一种运行时约束治理架构，将约束作为一等规范对象与状态、动作空间和奖励并列，通过预动作门控、动作时监控、事后审计和升级路由四个执行点实施硬约束，并扩展到多智能体工作流中的约束传播，以在工具调用执行前和执行中确保合规性。

- **创新类型：** 系统框架创新
- **创新性判断：** 池内文献 [3](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07728)（SARC）提出了运行时治理架构，但本文进一步引入约束传播机制和预动作门控的硬约束，与 [3](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07728) 形成差异化。

### 论文方向三：面向在线服务的分布式系统资源调度与性能建模

**拟定论文题目：** 面向在线服务的动态资源调度与配置无关性能建模方法

**论文核心问题：** 如何在在线服务场景中实现动态资源调度，同时通过配置无关的性能建模方法降低探索成本。

#### 创新点五：配置无关的冗余感知性能剖析

大多数现有性能剖析方法将操作集硬编码到特定配置，每次探索新配置时需从头重新剖析，成本高昂 [7](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07985)[21](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06914)。少数现有方法开始利用结构理解减少冗余剖析，但通常未充分利用模型配置值的跨模型复用性 [7](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07985)。为此，本文提出一种配置无关的冗余感知性能剖析方法，通过污点传播标记每个输入维度的来源，仅对请求相关维度进行单次扫描，利用模型配置值（如头大小、层数）的跨模型复用性，以在单次推理中覆盖所有配置变体，显著降低探索成本。

- **创新类型：** 方法设计创新
- **创新性判断：** 池内文献 [7](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07985)（Dooly）提出了配置无关的冗余感知剖析，但本文进一步引入污点传播和跨模型复用机制，与 [7](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07985) 形成差异化。

#### 创新点六：动态负载感知的在线资源调度

大多数现有资源调度方法依赖静态策略或固定阈值，难以适应在线服务中负载分布、性能目标和资源状态的持续变化 [21](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.06914)[24](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07547)。少数现有方法引入分层调度或在线反馈机制，但通常未将负载预测与调度决策形成统一闭环，导致调度滞后或震荡 [24](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07547)。为此，本文提出一种动态负载感知的在线资源调度框架，将实时监控、负载预测、配置生成和效果评估连接为统一闭环，通过短时与长时序列结合的负载预测指导调度决策，以增强在线服务在动态场景下的自适应能力。

- **创新类型：** 系统框架创新
- **创新性判断：** 池内文献 [24](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07547)（HAF）提出了分层调度框架，但本文进一步引入闭环反馈机制和短长时负载预测，与 [24](https://FAUST-BENCHOU.github.io/daily-arXiv-ai-enhanced/index.html?date=2026-05-11&paperId=2605.07547) 形成差异化。
